## Batch Notes

- These nodes are not present in `vuo-nodes/` or `docs/tixl-porting/PORT_STATUS_BOARD.md` at batch creation time.
- Primary output color follows TiXL string/value colors: string outputs use `ColorForString #779552`; int outputs use `ColorForValues #868C8D`.
- `SplitString` and `JoinStringList` are list-shaped B-lite nodes: Vuo `VuoList_VuoText` now carries `SplitString` fragments directly into `JoinStringList`, and the proof adapter renders both count and joined result.
- `StringLength` is listed as `A-lite` despite the namespace card's B row because the audited public contract is scalar `string -> int`; the semantic blocker is Unicode length parity, not Vuo list shape.
- All string tests should include empty input and newline behavior. Formatting nodes should additionally preserve TiXL's invariant-culture and `Invalid Format` behavior.
- Vuo CLI proof passed with compile/link/run return `0`; `loadedUserNodes` included all ten manufactured string node classes plus `my.string.batch.gradeAStringsProof`; screenshot visual check reported `mostlyBlack=false`.
