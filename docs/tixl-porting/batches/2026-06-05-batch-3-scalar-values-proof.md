## Batch Notes

- This batch intentionally upgrades existing manufactured nodes from `source contract test` to `composition proof`; it does not change their TiXL law.
- The proof adapter is proof-only and not a TiXL node. It renders 20 normalized rows from bool/int/float outputs so Vuo can visibly verify every node's output path.
- `my_PickBool` remains adapter-bounded: TiXL allows a `MultiInputSlot<bool>`, while the Vuo body currently exposes two bool inputs plus modulo index selection.
- `my_Div` and `my_Sqrt` keep their `NaN` source-contract tests; the Vuo visual proof uses finite values so the image adapter can render a stable scalar row.
