## Batch Notes

- `my_FloatsToList` preserves connected input order. Vuo receives the already-collected values as `VuoList_VuoReal`; this is value parity, not TiXL's exact reused output list instance.
- `my_FloatListLength` preserves null/empty behavior: both produce `0`.
- `my_PickFloatFromList` preserves TiXL `MathUtils.Mod` positive wrapping. Empty input returns `0.0`; negative index `-1` picks the last item.
- `my_SetFloatListValue` preserves value behavior for `Mode=Set/Add/Multiply`, wrapped nonnegative indices, `Index=-2` all-items, and no-op negative indices such as `-1`. When `TriggerSet=false` or the input list is empty, Vuo returns `previousResult` to represent TiXL's unchanged `Result.Value`.
- `my_SetFloatListValue` is a Vuo copy adapter. TiXL mutates and outputs the same `List<float>` reference; Vuo emits a copied list with the same values because Vuo list values should not expose TiXL reference aliasing as hidden mutable state.
- `ColorsToList` was not included because it is a `Vector4`/color-list node and belongs in a color/vector4 batch. `PickFloatList` was not included because it needs a list-of-lists/multi-list body-layer contract.
