## Batch Notes

- `my_IntListToFloatList` preserves null/empty -> empty and preserves list order. It uses `VuoReal` values as the Vuo body-layer float carrier; TiXL C# casts to 32-bit `float`, so exact single-precision rounding is not claimed here.
- `my_FloatListToIntList` preserves C# `(int)float` truncation toward zero for ordinary finite values. NaN, infinities, and out-of-32-bit-range values are not guarded in TiXL source and are outside this bounded Vuo proof.
- `my_PickFloatList` preserves TiXL positive modulo index wrapping across connected list inputs. When no connected list is declared (`inputCount <= 0`), it returns `previousResult` to represent TiXL's unchanged `Selected.Value`.
- `ComposeVec3FromList` was not included. Source audit shows it is list extraction + remap + optional stateful spring damping, not a plain list conversion. It should be handled in a stateful vec3/remap batch.
- First Vuo compile found a real body-layer bug in `my_PickFloatList`: assigning to const input `inputCount`. The fix sanitizes into a local `connectedCount`, then compile/link passed.
- Vuo CLI proof `batch-15-float-list-conversion-proof` compiled and linked with return code `0`, loaded all four custom nodes, opened an onscreen runner window, and produced Vuo-rendered PNG evidence at `artifacts/vuo_cli/batch-15-float-list-conversion-vuo-save.png`.
