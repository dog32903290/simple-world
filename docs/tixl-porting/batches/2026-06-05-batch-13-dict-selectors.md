## Batch Notes

- `my_SelectFloatFromDict` preserves TiXL's direct `Dict<float>.TryGetValue` selection. If the key is missing, output falls back to `previousResult`, representing the unchanged TiXL `Result.Value`.
- `my_SelectBoolFromFloatDict` preserves the strict threshold `floatValue > 0.5f`; `0.5` is false.
- `my_SelectVec2FromDict` preserves TiXL's sorted-key behavior: `SelectX` is X and the next sorted key is Y. It does not infer axis names.
- `my_SelectVec3FromDict` preserves `StringUtils.TryUpdateVectorKeysRelatedToX`: starting at `SelectX`, it accepts a three-key sorted run only when each key has the same length and differs from `SelectX` by at most one character.
- `JoinLists` was not included in this batch despite the namespace card's A grade. Source audit shows it operates on `StructuredList`; it needs a My World/TiXL data model contract before a real Vuo body should be claimed.
