## Batch Notes

- `my_AddInt2` preserves TiXL's component-wise `Width + Width`, `Height + Height` behavior.
- `my_Int2Components` preserves TiXL's raw aspect ratio division. Height `0` is not guarded in the Vuo body, matching the C# float division behavior rather than hiding a bad resolution.
- `my_MaxInt2` starts from `(0,0)` and takes component-wise maxima across the list, matching the TiXL loop.
- `my_ScaleResolution` uses C#-style integer truncation after multiplication. `ClampToValidTextureSize` clamps only to `1..16384`; despite the TiXL docs mentioning powers of two, the audited source does not round to powers of two.
- `my_ScaleSize` uses C#-style integer truncation after `InputSize * Scale * Stretch`.
- Vuo CLI proof `batch-9-int2-proof` compiled and linked with return code `0`, loaded all seven custom nodes, opened an onscreen runner window, and produced Vuo-rendered PNG evidence at `artifacts/vuo_cli/batch-9-int2-vuo-save.png`.
