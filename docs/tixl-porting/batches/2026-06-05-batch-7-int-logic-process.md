## Batch Notes

- `my_CompareInt` clamps mode to `0..3`, outputs both `IsTrue` and `ResultValue`, and defaults to equality mode.
- `my_PickInt` uses TiXL `MathUtils.Mod`, so negative indexes wrap forward. Empty Vuo list emits `0`; TiXL's multi-input dirty-flag behavior is adapter-bounded.
- `my_ClampInt` follows `MathUtils.Clamp`: `min(max(value, min), max)`. It does not sort reversed ranges.
- `my_FloatToInt` matches C# cast behavior, truncating toward zero.
- `my_GetAPrime` uses TiXL's 1-based index; index `< 1` returns `-1`.
- `my_MaxInt` and `my_MinInt` preserve TiXL empty-input sentinels as `Int32.MinValue` and `Int32.MaxValue`.
